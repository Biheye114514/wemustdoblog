# 个人博客后续规划

这个博客已经完成了从本地模板到线上部署的第一步。后续可以按照“内容建设 -> 视觉优化 -> 功能扩展 -> 长期维护”的顺序推进。

## 阶段一：先把博客变成自己的

当前最重要的是让网站内容从模板状态变成真正属于自己的博客。

建议先做：

1. 删除模板文章：

   ```txt
   src/content/blog/astro-content-collections.md
   src/content/blog/design-engineering.md
   ```

2. 保留并完善第一篇自己的文章：

   ```txt
   src/content/blog/hello-blog.md
   ```

3. 再写 2-3 篇短文：

   - 最近在学什么
   - 最近看完的一部番
   - 搭建这个博客踩了哪些坑

4. 修改“关于我”页面，让它更像自己，而不是模板介绍。

## 阶段二：整理栏目和标签

建议先固定几个长期使用的标签：

```txt
日常学习
番剧感想
技术交流
踩坑记录
项目复盘
碎碎念
```

以后每篇文章都尽量从这些标签里选择，博客会更有秩序，也方便后续扩展分类页或专题页。

## 阶段三：优化外观

内容稳定后，再逐步调整视觉体验。

可以依次优化：

1. 头像、favicon、网站标题
2. 首页文案和入口卡片
3. 文章卡片样式
4. 背景图或背景动效
5. 移动端显示效果

建议不要一次性大改。先让网站稳定地像自己的博客，再慢慢加细节。

## 阶段四：添加自定义板块

等内容有一点积累后，可以考虑加这些板块：

### 番剧专区

记录番名、观看状态、评分、短评和推荐理由。

### 学习日志

按时间记录最近学了什么，适合写短内容和阶段性复盘。

### 技术踩坑

专门记录报错、解决方案、配置过程和开发经验。

### 友链

以后可以放朋友的网站、喜欢的博客或常用资源。

### 时间线

记录博客上线、改版、学习节点和一些重要变化。

## 阶段五：养成更新流程

每次本地改完文章、样式或配置后，按这个流程发布：

```powershell
npm run build
git add .
git commit -m "Update blog"
git push
```

推送到 GitHub 后，Netlify 会自动重新部署。

## 下一步建议

最适合现在立刻做的是：

1. 删除模板文章。
2. 写一篇《搭建这个博客踩了哪些坑》。
3. 把这次搭建过程中遇到的内容记录下来，比如 npm 依赖、Git 提交、GitHub 推送、Netlify 部署。

这篇文章最容易写，因为它来自真实经历，也会成为以后回头查看的第一份技术记录。
